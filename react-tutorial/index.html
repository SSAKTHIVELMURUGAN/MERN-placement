Difference Between Semantic and Non-Semantic Tags in HTML

Aspect Semantic Tags Non-Semantic Tags
Definition Tags that clearly describe their meaning and content. Tags that do not provide information about their content.
Examples
  • <header>
  • <nav>
  • <article>
  • <section>
  • <footer>
  • <div>
  • <span>
Usage Helps in understanding the structure and meaning of the content. Used for generic containers when semantic meaning is not needed.
Accessibility Improves accessibility for assistive technologies. Less helpful for accessibility purposes.
SEO Helps search engines better understand the content. Does not aid in SEO as much as semantic tags.

Difference Between Internal CSS and External CSS

Aspect Internal CSS External CSS
Definition CSS styles defined within the <style> tag in the HTML document. CSS styles defined in a separate .css file linked to the HTML document.
Location Inside the <head> section of the HTML document. In a separate file, typically with a .css extension.
File Management Styles are embedded directly in the HTML file. Styles are placed in an external file, separate from the HTML.
Use Case Suitable for small projects or single-page documents. Suitable for larger projects or sites with multiple pages.

Attributes in HTML

  • id Attribute: Uniquely identifies a single HTML element.
  • class Attribute: Assigns one or more classes to an element, allowing for shared styling.
  • name Attribute: Used to reference form elements during data submission.
  • data Attributes: Used to store custom data private to the page or application.
  • Global Attributes: Attributes that can be used on any HTML element, such as style, title, lang, dir, hidden, and tabindex.
